20.  Security Protocols

Daily backups: Our database is backed up nightly to ensure we can safely restore user progress in the event of an emergency.

Manual Pen Testing: We actively attempt to hack our own product on a weekly basis to look for vulnerabilities. We do so to detect methods of abusing the credit system, to test accessing restricted data, and to evaluate the security of our APIs from direct attack.

Workspace Access Control: Workspaces have restricted access by default, which restricts anyone from loading any details related to a flow unless they have been added to the workspace and explicitly granted access.

Static Code Analysis: We’ve undergone rigorous scrutiny by CASA (Cloud Access Security Assessment) to evaluate our cloud applications and infrastructure for potential security risks, ensuring compliance with industry standards and best practices for data protection.

3rd Party Auth Restriction: No user can ever access another user's personal credentials under any circumstances. This makes it more difficult to share flows but is a security decision we enforce to establish a firm foundation for credential protection.


We are committed to following necessary Security Protocols per the EU-U.S. Data Privacy Framework (EU-U.S. DPF) and the UK Extension to the EU-U.S. DPF, and to the rights of EU and UK individuals.
